> проверяемую доказательную структуру: каждый вывод привязан к документу,
> странице и условиям применимости.

## Что мы построили

**Evidence-first GraphRAG-платформа** для работы с научно-техническими знаниями
горно-металлургической отрасли. По вопросу на естественном языке (включая
числовые условия вида «сульфаты 200–300 мг/л, сухой остаток ≤1000 мг/дм³»)
система возвращает единый структурированный ответ:

- **Evidence-таблицу** — фрагменты первоисточников, каждый с `документ + страница + год + география`;
- **Граф знаний** — сущности (материалы, процессы, оборудование, параметры) и типизированные связи (`INCREASES / REDUCES / USES / HAS_PARAMETER …`), где **каждое ребро несёт ссылку на источник**;
- **Противоречия** — автоматически найденные конфликтующие утверждения из разных документов (с парой источников);
- **Пробелы знаний** — где данных нет или мало (`knowledge_gap`, `missing_combination`, `missing_numeric_data`…);
- **Краткий вывод** с уровнем достоверности и предупреждениями;
- **Экспортируемый отчёт** (Markdown/JSON) со всеми ссылками.

## Архитектура

```
React 18 + Vite + Tailwind + React Flow  (frontend/, SPA)
        │  same-origin /api/*
FastAPI «app»  — auth, upload, dashboard, export, раздача UI
        │  app/rag_bridge.py (единый мост, fallback на demo-данные)
RAG-движок «rag» — GraphRAG-пайплайн, чистый stdlib (sqlite3 + FTS5)
        │
index.sqlite (5.8 ГБ) — FTS5 BM25 + relations + parameters
```

**Пайплайн запроса:** разбор запроса (сущности по 44k-словарю + числовые
условия детерминированным парсером, без LLM) → гибридный поиск (BM25 ⊕
entity-expansion, слияние RRF) → метаданные-фильтры (год/география/тип через
join documents) → числовой фильтр по structured-параметрам → сборка evidence →
граф из relations-индекса → противоречия + пробелы → ответ.

**Ключевые инженерные решения:**
- Гигантские JSONL (2–3 ГБ) **никогда не грузятся в RAM** — только стриминг при
  сборке индекса и SQLite-запросы при работе; RAM плоская, ответ — миллисекунды.
- MVP-путь **без тяжёлых зависимостей**: никаких Qdrant/Neo4j/langchain —
  sqlite3 из стандартной библиотеки + FastAPI для API.
- **Контракт `SearchResult` заморожен**: фронтенд не зависит от режима движка
  (реальный индекс ↔ demo-fallback переключаются прозрачно, форма ответа одна).
- Числовой парсинг детерминированный (regex экстрактора), не LLM — воспроизводимо.
- Ремонт мojibake на индексации; источник неотделим от каждого элемента ответа
  (тест падает, если `sourceName/page` потерялись).

## Демо-сценарии (все три проходят на полном индексе)

1. **Обессоливание воды / числа** — условия «200–300 мг/л» и «≤1000 мг/дм³»
   распознаются, numericMode = **structured**, confidence high, 15 evidence с
   привязанными параметрами.
2. **Циркуляция католита / граф + противоречия** — граф с читаемыми узлами
   (католит, электролиз, катод…), 50/50 рёбер с источниками; найдено
   **2 противоречия** (`INCREASES` в 07_2003_cm.pdf vs `REDUCES` в CM_06_15.pdf).
3. **Au/Ag/МПГ за последние 5 лет** — «за последние 5 лет» → фильтр
   `2021–2026`, все источники в окне (2024–2025).

## Функциональность поверх поиска

- **Логин/роли** (демо-авторизация), рабочая область исследователя;
- **Загрузка своего документа** → статус обработки → извлечение;
- **Карточка извлечения по любому документу корпуса** — сущности по типам,
  числовые параметры, типизированные связи, выводы (реальные данные из индекса);
- **Карточки источников** (тип, год, география, надёжность, фрагмент);
- **Дашборд** со статистикой корпуса; **Research Memory**; экспорт отчётов.

## Запуск

```bash
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
.venv/bin/python run.py --port 8011        # Windows: py run.py
```

- UI (React): http://127.0.0.1:8011/ — вход: `researcher@demo.local / demo123`
- Swagger: http://127.0.0.1:8011/docs · старый demo-UI: `/demo`
- Если `rag/data/index.sqlite` отсутствует — система честно работает в
  demo-режиме (mock), форма API не меняется. Пересборка индекса:
  `python -m rag.index build --data-root <распакованный nornikel> --db rag/data/index.sqlite`

Пересборка фронтенда: `cd frontend && npm ci && VITE_API_BASE_URL=/ npm run build`.

## Честные ограничения

- Противоречия — эвристика (противоположные типы связей между теми же
  сущностями в разных документах); помечаются как требующие проверки.
- Vector-канал поиска — опциональный (интерфейс готов, по умолчанию выключен);
  ранжирование сейчас BM25 + RRF.
- В части текстов корпуса остался мусор кодировки из исходной экстракции —
  чинится на стороне экстрактора.
