# Что сделано (кратко, для организаторов)

**Команда:** Закирна · трек «Научный клубок», задача 2

## Результат

Работающая evidence-first GraphRAG-система: вопрос на естественном языке →
структурированный ответ, где **каждое утверждение привязано к документу и
странице** (evidence-таблица, граф знаний, противоречия, пробелы, отчёт).
Единое приложение: React UI + FastAPI + RAG-движок, один порт.

## По шагам

1. **RAG-движок** (`rag/`, чистый stdlib): SQLite FTS5-индекс по полному корпусу —
   **441 714 чанков, 2.68 млн связей, 696 тыс. числовых параметров, 2 009
   документов**. Гигантские JSONL (до 3 ГБ) обрабатываются стримингом, RAM плоская.
2. **Гибридный поиск**: BM25 ⊕ расширение по сущностям (44k словарь) → RRF →
   фильтры (год/география/тип) → **structured-фильтрация по числовым условиям**
   запроса («сульфаты 200–300 мг/л» понимается детерминированным парсером, не LLM).
3. **Граф знаний**: типизированные связи (INCREASES/REDUCES/USES…), каждое ребро
   с источником; читаемые имена узлов (католит, электролиз, катод…).
4. **Аналитика**: автопоиск противоречий между документами (с парой источников)
   и пробелов знаний; краткий вывод с уровнем достоверности.
5. **Backend-API** (FastAPI, 20 эндпоинтов): поиск, граф, источники, извлечение
   по документу, загрузка документов, дашборд, экспорт отчётов, демо-авторизация.
   Захардкоженные демо-эндпоинты заменены на реальные данные из индекса.
6. **Frontend** (React 18 + Vite + React Flow) интегрирован same-origin: бекенд
   раздаёт SPA с корня, API без CORS; старый demo-UI сохранён на `/demo`.
7. **Проверка**: 14/14 тестов движка, сквозной прогон 19 интеграционных проверок
   всех эндпоинтов UI, все **3 судейских демо-сценария проходят** на полном
   индексе (числовой, граф+противоречия, временной фильтр).

## Запуск

```bash
pip install -r requirements.txt && python run.py    # http://127.0.0.1:8000
```

Вход: `researcher@demo.local / demo123` · Swagger: `/docs`.
Без 5.8 ГБ индекса система работает в demo-режиме; с `rag/data/index.sqlite` —
полный RAG. Подробности: [README.md](README.md), [docs/SETUP.md](docs/SETUP.md).
